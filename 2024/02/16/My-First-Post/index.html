強化式學習

  1. 問題 => 需要解決的任務
  2. 解決方法 => 完成以上任務所設計的演算法
    *將深度學習作為解決方法的演算法，就稱為深度強化式學習

特性

  1. 不需要具有label的資料
    => 不需要告知演算法正確答案，只需給予最終目標或大方向
    => 透過reward來告知該做甚麼事

    讓演算法不斷進行嘗試，從錯誤中學習，直到能達成目標並或得獎勵為止

  2. 最終極目標是最大化reward
    => 需要知道與任務有關的基礎知識
    => 目標為得到最多正向的reward
    => 負的reward稱為懲罰

訓練流程: 演算法做出決策後會產生一個reward來告知演算法應該強化或是懲罰該決策，並不斷重複以上動作直到達成目標為止

方法

1. 動態規劃 dynamic programming

核心概念: 將複雜問題拆解成一個個較為簡單的子問題直到子問題可以用目前的資訊解決為止
缺點: 現實中無法獲得任務中的所有資訊，因此很難分解問題

2. 蒙地卡羅法 Monte Carlo methods

核心觀念: 從環境中隨機取樣，不斷的try and error

現實中會將兩種方法結合，以動態規劃完成簡單目標，再以try and error的方式應付其餘情況

強化式學習架構

將目標整合成損失函數loss function

=> 給予資料後得知與目標的距離有多遠

環境是動態的，依照時間切分成不同的狀態state

=> 環境等同於輸入的資料如伺服器溫度、電費等
=> 不同的時間，環境變數也會不同，形成不同的狀態

動作Action

=> 強化式學習需要基於目前環境作出最佳的決策後執行動作，並且會對未來產生影響

Reward

=> 當一個動作結束後，演算法會得到一個reward，告知自己在達成目標的表現如何

Agent

=> 目標是讓reward達到最佳
=> 深度強化式學習會將代理人設定為使用深度學習演算法
:::info
使用deep learning algorithm作為Agent的演算法原因:
原始方式是利用建立查詢表儲存所有state的資訊，但是隨著環境變複雜，查詢表大小會太大，因此使用深度學習的好處在於可以只記住每個狀態的關鍵特徵(feature)，減少參數的使用量並降低複雜度
:::

模型畫強化式學習問題: 馬可夫決策過程

多臂拉霸機問題 multi-arm bandit

眼前有10台拉霸機，每一台拉霸機最高獎金是10元，且每台機器的平均獎金不同，如何才能選到平均獎金最高的拉霸機?

探索(exploration)和利用(exploitation)

探索 => 隨機嘗試不同機台，觀察不同機器的期望獎金(依據過去執行該動作得到的平均獎金)
利用 => 累積經驗後，可以推論出期望獎金最高的機台，而且只選擇該機台來玩

現實中，會兩種方法會一起使用但比例不同

貪婪方法(greedy method)

原理: 結合探索與利用兩種方法，加入機率值ε，也就是說有ε的機率會去探索(隨機選擇一台機器)，1-ε的機率會做利用(選擇目前表現最好的機台)的動作

Softmax策略

原理: 可以先知道每個動作(action)的成功率有多高，因此在隨機選擇動作的時候能夠避免選到非常糟糕的選項
公式:
P(k): 動作集合k的每個動作機率分布
Q(k): 動作的價值陣列(輸入的參數)
τ: 稱爲“溫度”，τ越小代表每個動作之間的機率值差異變大，則平均獎賞高的動作被選取的概率越高。τ趨近於0時Softmax算法趨於僅利用，τ趨於無窮大時Softmax算法趨於僅探索

利用拉霸機問題的演算法優化廣告推送策略

狀態、動作與reward

  • 狀態空間: 記作S，環境中所有狀態的集合
  • 動作空間: 記作A，某個狀態中所有可能動作的集合
  • Reward: 執行某個動作後獲得的回饋

    state-action pair (s, a)

馬可夫決策過程 Markov decision process

馬可夫性質 => 過去的狀態資訊不會影響未來演算法作出決策
任何符合馬可夫性質的控制任務皆可稱為馬可夫決策過程

策略與價值函數

強化式學習的終極目標是要讓reward最大化，而會產生reward是因為狀態發生改變

策略函數 policy function(π)

Agent針對某個state所採取的對策
能針對某個state算出每個動作為最佳可能的機率，並在下次更新策略時會儘量多採取最佳的動作

最佳策略 optimal policy => 若一個策略能持續使環境給出最大的回饋值，該策略為一個state的optimal policy

價值函數 value function => Q函數

輸入為state， 輸出為在目前的state下，利用策略π執行某個action產生的reward，更新完之後在去決定下一個策略

若將deep learning algorithm當作Q function，則稱為 deep Q-learning(deep Q-Network)
